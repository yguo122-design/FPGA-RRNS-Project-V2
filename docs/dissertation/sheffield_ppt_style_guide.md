# Sheffield-Themed Dissertation Defence PPT Style Guide

## 1. Overall Design Goal

The presentation should look:
- academic and formal
- clean and modern
- technically credible
- visually consistent with The University of Sheffield identity

Avoid overly decorative or commercial-looking templates. The tone should be **professional, restrained, engineering-oriented, and suitable for a final-year dissertation defence**.

---

## 2. Logo Usage Specification

### Recommended logo file
Use the vector logo:
- figure/sheffield-logo.svg

Alternative logo asset:
- figure/logo.svg

### Placement rules
- **Title slide**: place the Sheffield logo in the **top-left or top-right corner**, medium size.
- **Content slides**: place a smaller logo in the **top-right corner** or in the **footer area**.
- **Q&A slide**: logo may be slightly enlarged for a polished closing page.

### Logo sizing
- Title slide: about **1.8-2.4 cm** high
- Content slide: about **0.8-1.2 cm** high

### Logo usage rules
- Keep clear space around the logo.
- Do not stretch, rotate, recolour, or apply shadows/glows.
- On dark purple backgrounds, use the white version if available.

## 3. Recommended Colour Palette

The local logo asset uses the University-style purple:
- **Primary Sheffield Purple**: #440099

Use the following palette for the whole deck:

### Main palette
- Primary brand purple: **#440099**
- Secondary purple: **#6A2DBA**
- Light lavender background: **#F3ECFF**
- Deep charcoal text: **#222222**
- Mid grey for helper text: **#666666**
- White background: **#FFFFFF**

### Accent colours for charts
Use only 1-2 additional accents:
- Highlight accent: **#00A3A3**
- Warm emphasis accent: **#C98A00**

### Colour rules
- Most slides should use **white background + purple headings**.
- Use purple blocks or divider lines for emphasis.
- Avoid excessive colour variety.
- Do not use bright red/green unless needed for technical contrast.

## 4. Font Recommendations

### English text
Preferred fonts:
- Aptos
- Segoe UI
- Arial
- Calibri

### Chinese text if needed
Preferred fonts:
- Microsoft YaHei
- DengXian
- Source Han Sans

### Font hierarchy
- Slide title: **28-34 pt**, bold
- Section subtitle: **20-24 pt**
- Body text: **16-20 pt**
- Figure caption / note: **11-14 pt**

### Typography rules
- Use **one sans-serif family consistently**.
- Avoid mixing many font styles.
- Keep text left-aligned except title page.
- Use bold only for key phrases, not for full paragraphs.

---

## 5. Page Layout Suggestions

### General layout principle
Each slide should communicate **one core message only**.

Use this structure:
1. short title
2. one chart / one diagram / one key comparison
3. 2-4 bullets summarising the message

### Margin and spacing
- Leave clear white space around all objects.
- Avoid filling the entire slide.
- Keep top and bottom margins consistent.
- Align text boxes and figures to a simple grid.

### Recommended layout patterns

#### A. Title slide
- Large title in the centre-left
- Candidate name, student ID, supervisor below
- Sheffield logo at top corner
- Clean background, no heavy decorations

#### B. Problem / objective slide
- Left side: bullets
- Right side: one small conceptual illustration or icon block

#### C. Architecture / methodology slide
- Large central figure
- 2-3 concise bullets below or on the side

#### D. Results slide
- One chart per slide preferred
- If two charts are needed, use a balanced 2-column layout
- Add one highlighted takeaway box

#### E. Conclusion slide
- 3 key conclusions only
- Use icons/check marks or numbered emphasis blocks

#### F. Appendix slide
- Simplified layout, smaller title, compact figure + brief explanation

---

## 6. Chart Beautification Rules

Because this dissertation is data-heavy, chart styling is very important.

### Chart style rules
- Increase line thickness to improve projection visibility.
- Use fewer grid lines.
- Enlarge axis labels and legends.
- Keep legend placement consistent.
- Highlight the best-performing algorithm using the main purple or a strong accent.

### Suggested chart emphasis mapping
- C-RRNS-MLD: dark purple
- RS: teal or dark gold
- 3NRM-RRNS: medium purple
- 2NRM variants: lighter blue / lavender

### Chart readability rules
- Minimum axis / legend font: **16-18 pt**
- Use short labels where possible
- Remove cluttered annotations
- Ensure figures are readable on a projector from a distance

## 7. Suggested Visual Style for This Specific Defence

### Tone
The presentation should feel like:
- an engineering research defence
- trustworthy and evidence-based
- elegant but not flashy

### What to emphasise visually
The designer should especially strengthen these pages:
- project motivation and research gap
- FPGA platform architecture
- probabilistic fault injection innovation
- BER comparison results
- latency/resource/storage trade-off summary
- final conclusions and application recommendations

### Most suitable visual style
- white background
- purple title bar or thin top banner
- consistent footer with page number
- clean tables with light grey borders
- subtle use of lavender highlight blocks

---

## 8. Per-Slide Beautification Advice

### Title slide
- Add Sheffield logo and a strong title hierarchy.
- Include candidate name, student ID, supervisor, and date.
- Use a formal university feel.

### Motivation slide
- Use a 2-column layout with one side for the fault problem and one side for the research gap.

### Objective slide
- Convert the objectives into 3-4 visual blocks instead of a long list.

### Platform advantages slide
- Use icon-based cards for fairness, low BRAM overhead, scalability, and cross-validation.

### Architecture slide
- Keep the system diagram large and readable.
- Add a short purple takeaway box: “PC configures, FPGA executes full BER sweep autonomously.”

### Results slides
- Keep only the figure and 2 bullet takeaways.
- Avoid dense paragraphs.
- Highlight the most important trend in a coloured callout box.

### Quantitative comparison slide
- Use a clean table with the best values bolded or coloured.
- Add a small summary strip at the bottom: reliability / speed / compactness.

### Conclusion slide
- Present the 3 most important conclusions as numbered blocks.

### Q&A slide
- Use the logo, a short thank-you line, and a calm uncluttered background.

---

## 9. Animation and Transition Rules

- Use **minimal animation only**.
- Recommended: Fade or Appear.
- Do not use spinning, bouncing, or excessive motion.
- Keep transitions subtle and consistent.

---

## 10. Final Quality Control Checklist

Before delivery, the third party should ensure:

- logo is correctly placed and proportioned
- entire deck uses one unified colour system
- all charts are readable at projection distance
- no slide is overcrowded
- line spacing and alignment are consistent
- title style is uniform across all pages
- conclusion pages are visually stronger than body pages

---

## 11. One-Sentence Creative Direction for the Designer

Please redesign this dissertation defence PPT in a **clean academic engineering style using The University of Sheffield purple (#440099) as the primary brand colour, with consistent logo placement, strong chart readability, restrained elegance, and clear one-message-per-slide layout**.
