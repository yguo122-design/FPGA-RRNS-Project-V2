## [TIMING FIX] decoder_2nrm.v — 3-Stage Pipeline to Resolve WNS = -28.829 ns

**Date:** 2026-03-16  
**Severity:** Critical (Implementation failure, 981 failing endpoints)  
**File Modified:** `src/algo_wrapper/decoder_2nrm.v`  
**Comment Update:** `src/ctrl/auto_scan_engine.v` (pipeline latency comment only)

### Problem

Vivado Implementation failed with:
- **WNS:** -28.829 ns
- **Failing Endpoints:** 981
- **Critical Path:** `u_fsm/u_engine/inj_out_a_latch_reg[13]/C` → `u_fsm/u_engine/u_dec/a/u_dec_2nrm/ch9/distance_reg[0]/D`
- **Logic Levels:** 52

The `decoder_channel_2nrm_param` sub-module contained ~52 LUT levels of pure
combinational logic in a single clock cycle:
1. CRT reconstruction: MUX → diff_mod → coeff_mod → x_cand → x_cand_16 (~33 LUT)
2. Six modular reductions: x_cand_16 % modulus[k] for k=0..5 (~19 LUT)
3. Hamming distance popcount adder (~5 LUT)

### Fix

Split the single-cycle combinational chain into **3 registered pipeline stages**
inside `decoder_channel_2nrm_param`:

| Stage | Cycles | Logic | New Registers |
|-------|--------|-------|---------------|
| Stage 1 | 0→1 | CRT reconstruction | `x_cand_16_s1`, `r0_s1..r5_s1`, `valid_s1` |
| Stage 2 | 1→2 | 6× modular reduction | `cand_r_s2[0..5]`, `recv_r_s2[0..5]`, `x_cand_16_s2`, `valid_s2` |
| Stage 3 | 2→3 | Hamming distance | `x_out`, `distance`, `valid` (existing, repurposed) |

**Estimated LUT depth per stage after fix:**
- Stage 1: ~18 LUT levels (CRT only)
- Stage 2: ~8-10 LUT levels (each modulo independent, parallel)
- Stage 3: ~5-8 LUT levels (6-input popcount)

### Latency Change

| Module | Before | After |
|--------|--------|-------|
| `decoder_channel_2nrm_param` | 1 cycle | 3 cycles |
| `decoder_2nrm` total | 2 cycles | **4 cycles** |
| `decoder_wrapper` total | 2 cycles | **4 cycles** |

### Upper-Level Impact

**No functional changes required** in any other module:
- `auto_scan_engine.v` `DEC_WAIT` state polls `dec_valid_a && dec_valid_b`
  (not a fixed counter), so the 2-cycle increase is absorbed automatically.
- Watchdog threshold is 10,000 cycles — far above the new 4-cycle latency.
- `decoder_wrapper.v` interface unchanged.
- `result_comparator.v` waits for `valid_in` — unaffected.

### Key Design Decisions

1. **Received residues re-registered at Stage 1**: `r0_s1..r5_s1` are registered
   alongside `x_cand_16_s1` to keep them time-aligned for Stage 2 comparison.
2. **`x_cand_16` carried through Stage 2**: `x_cand_16_s2` is forwarded to
   Stage 3 so the final `x_out` register receives the correct value.
3. **All new registers use `rst_n` async reset** and are driven by `clk`.
4. **Algorithm logic unchanged**: only timing structure modified.
